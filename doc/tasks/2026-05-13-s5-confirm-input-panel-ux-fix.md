# DEV2-S5.11 Confirm Input Panel UX Fix

日期：2026-05-13
开发者：dev2

## 背景

用户反馈系统操作确认面板仍像普通聊天：输入 `1` 会被额外回显，面板位置和输入状态不清晰，并且仍有英文确认文案。产品要求是完全新手也能用：看中文选项，输入框确认，系统执行并给结果。

## 完成内容

- 系统权限面板中文化：命令、目录、操作、原因、影响、下一步选择。
- REPL 增加动态确认输入 prompt：确认态下显示 `选择 1 允许 · 2 允许并记住 · 3 取消 >`。
- 系统确认输入不再打印额外 `◇ 1` 聊天回显。
- 确认态输入非法内容时本地提示，不进入 AI 对话。
- 更新 system-approval 单测和 REPL 冒烟断言。

## 验收命令

- `npm run build`
- `npm run test -- system-approval`
- `npm run smoke:repl`

## 后续

真正的“底部固定面板”需要进入统一 Choice Panel/TUI 层实现，避免每种确认都单独打印。当前版本先修复系统操作确认的用户混淆和重复回显问题。
