# S20.2 等待体验优化

日期：2026-05-15
负责人：dev1（全部独立）
优先级：🔴 P0
估时：3h

## 问题

从输入到 AI 首字节响应，用户只能看到静止 spinner "思考中"，没有计时、没有进度感，5-15 秒的等待感觉很"卡死"。

## 实现

### 文件: `src/cli/repl.ts`

**三阶段状态机:**

```
idle → connecting → waiting → streaming → idle
```

**阶段 1 — connecting (0-1s):**
```
  ◉ 正在连接 DeepSeek API...
```

**阶段 2 — waiting (1-15s):**
```
  ◉ AI 分析中 [4.7s]  ██████░░░░░░░░░░░░
```

- 计时器：`startTime = Date.now()`，每 1s 更新 `[N.Ns]`
- 脉冲动画：帧序列 `['◉','◔','◑','◕']`，每 80ms 切换
- 超过 10s 追加提示行
- 进度条：`█` + `░` 组合，位置随机波动模拟"活跃感"

**阶段 3 — streaming:**
```
  ◉ 输出中 [8.1s]  已接收 847 tokens
```

- Token 计数：从 stream chunk 中估算 (约 3.5 字符 = 1 token)

### 文件: `src/cli/theme.ts`

新增辅助函数：
- `waitingSpinner(elapsed: number): string` — 阶段 2 渲染
- `streamingStatus(elapsed: number, tokens: number): string` — 阶段 3 渲染

### 验收

```
测试: 阶段 1→2→3 状态转换正确
测试: 计时器精度 ±0.2s
测试: Ctrl+C 可中断所有阶段
测试: streaming 完成回到 idle
```
