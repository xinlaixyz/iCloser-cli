# 任务驾驶舱缺陷排查与修复记录

日期：2026-05-22  
状态：持续优化  
范围：REPL Mission Cockpit、Golden Path、非代码任务结果展示

## 1. 本轮发现的问题

### D1：投资报告被错误展示为项目分析

真实输入：

```text
你的投资报告分析太少了
```

旧驾驶舱显示：

```text
类型    项目分析
目标    读项目结构、找风险、给出可执行优化路径
```

问题：

- 用户问的是投资报告，不是代码项目结构。
- 驾驶舱一开始就给错任务画像，会让用户觉得 AI 跑偏。

修复：

- `missionProfile()` 增加投资/市场研究画像。
- 命中关键词：投资、融资、估值、市场、商业、竞品、尽调等。
- 新显示：

```text
类型    投资 / 市场研究
目标    围绕公司、市场、融资、竞品和风险做证据化分析
执行顺序  明确研究问题 → 搜索公开资料 → 抓取关键来源 → 归纳投资判断 → 列出尽调缺口
```

### D2：非代码任务结果被说成“没有代码补丁”

旧结果面板：

```text
交付    本轮没有代码补丁
```

问题：

- 对投资报告、网页访问、项目分析来说，没有代码补丁是正常的。
- 这会把“已交付报告”误导成“没有交付”。

修复：

- `renderMissionResult()` 按任务类型生成交付文案。
- 非代码任务显示 `已生成 N 行结果`。
- 代码任务仍显示补丁文件。

### D3：非代码任务验证文案错误

旧结果面板：

```text
验证    等待写入后验证
```

问题：

- 投资报告和网页研究没有“写入后验证”。
- 对这类任务，验证应该是“基于工具证据核对”。

修复：

- `summarizeVerification()` 按任务类型展示。
- 非代码任务有工具证据时显示：

```text
验证    已基于 N 条证据核对
```

### D4：Golden Path 只把命令/Git 当验证

旧逻辑：

- 只有 `run_command` 或 `git_status` 才算验证。
- 网页、投资、市场分析即使有 20 条网页证据，也显示验证不足。

修复：

- `runAgentTaskLoop()` 改为：
  - 代码/启动任务：仍以命令、Git、运行证据为验证。
  - 网页/分析/通用任务：成功工具证据即可作为验证证据。

### D5：开始面板没有显示工具预算

旧问题：

- 用户只能看到 AI 即将执行什么步骤。
- 看不到本轮是否有限流、是否会压缩证据、是否会控制 web_search 数量。
- 真实投资报告任务出现 20 次工具调用和 45K tokens 后，用户会感觉 AI 在失控搜索。

修复：

- `missionProfile()` 增加 `budget` 字段。
- 任务开始面板新增：

```text
预算    每轮最多 8 次工具 · web_search ≤5 · 证据压缩后合成
```

不同任务类型显示不同预算策略：

- 投资/市场研究：工具限流、web_search 限流、证据压缩。
- 代码交付：影响面定位、关键文件去重、补丁后验证。
- 网页访问：优先目标 URL，追问复用上一轮网页证据。
- 项目启动：先读构建配置，再执行命令，后台进程记录 PID。

### D6：结果面板不展示关键来源

旧问题：

- 结果面板只显示工具名，例如 `web_search, web_fetch`。
- 用户不知道证据来自 36 氪、官网、PitchBook 还是其他来源。
- 这会削弱投资报告、网页研究和市场分析的可信度。

修复：

- `renderMissionResult()` 增加 `evidenceTargets`。
- REPL 从 Evidence Store 提取工具目标并传入结果面板。
- URL 自动归一化为域名，路径/查询保留简短文本。

新结果示例：

```text
来源    pitchhub.36kr.com, icloser.xyz, iCloser 公司 融资 投资 估值
```

## 3. 新增测试

文件：`tests/mission-cockpit.test.ts`

新增覆盖：

- 投资报告任务必须显示 `投资 / 市场研究`，不能显示 `读项目结构`。
- 非代码分析结果不能显示 `本轮没有代码补丁`。
- 非代码分析结果不能显示 `等待写入后验证`。
- 有 20 条证据时，结果面板必须显示 `已基于 20 条证据核对`。
- 投资/市场研究开始面板必须显示 `web_search ≤5` 和 `证据压缩`。
- 非代码结果面板必须展示关键来源域名，例如 `pitchhub.36kr.com`、`icloser.xyz`。
