## 任务详情

### 1. `ic rollback --auto` (3h)

**现状**：Auto-8 已实现写入前快照（`.icloser/snapshots/<task-id>/`），但回滚需手动触发 `ic rollback <task-id>`。

**改动**：在 executeTask 的验证失败路径（3轮修复后仍失败），自动执行回滚。

**验收**：模拟验证失败 → 文件自动恢复为修改前状态。

### 2. 跨会话任务恢复 (4h)

**现状**：REPL 退出后，`state.conversation` 和 `state.context` 丢失。下次启动是全新会话。

**改动**：退出时保存 `state` 到 `.icloser/session.json`。启动时检测并提示恢复。

**验收**：REPL 中创建任务 → 退出 → 重启 → 提示恢复 → 任务继续。

### 3. 真实验收测试 (4h)

**现状**：498 测试全用 mock provider。无法检测真实 AI 编排质量。

**改动**：`live-acceptance.mjs` 增加 `--live` 模式。需要真实 API key，运行 5 场景并对比 mock vs real 输出。

**验收**：设置 `ANTHROPIC_API_KEY` → `npm run acceptance -- --live` → 5/5 通过。

### 4. 增量索引刷新 (6h)

**现状**：文件写入后，上下文不变。修复循环中 AI 可能拿到旧文件内容。

**改动**：在 Auto-5 测试生成后、修复循环开始前，调用 `scanner.ts` 的增量扫描更新索引。

**验收**：多文件修改任务 → 第二个文件生成时引用第一个文件的更新内容。

### 5. 自然语言对话式澄清 (8h)

**现状**：`ic t "添加登录"` 直接执行，不追问"什么类型的登录？"

**改动**：在 `generateExecutionPlan` 前增加澄清步骤。如果任务描述模糊（无具体文件/函数名），AI 先返回澄清问题列表。

**验收**：`ic t "添加功能"` → AI 回复"请明确：1) 什么功能 2) 哪个模块 3) 预期行为"

### 6. 覆盖率 48%→60% (8h)

**现状**：覆盖率 43.6%，CLI 层几乎无覆盖。

**改动**：为 `ic gen`、`ic code`、`ic plan` 命令处理函数增加单元测试。为 `repl.ts` 核心函数增加测试。

**验收**：`npm run test:coverage` 显示 ≥60%。
