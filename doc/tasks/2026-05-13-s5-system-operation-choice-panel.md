# S5.3 System Operation Choice Panel

## 背景

用户明确指出：输入框和系统权限操作要从“用户输入命令”的角度转为“用户选择”的角度。系统应像截图里的确认界面一样，先解释即将执行什么，再让用户按数字确认。

## 产品原则

- 用户表达目标，系统整理可执行动作。
- 用户不需要记命令，也不需要理解 shell。
- 系统权限操作必须先展示中文确认面板。
- 用户只做选择：执行一次、取消；后续可扩展“执行并记住此选择”。

## 本次实现

- `src/cli/repl.ts`
  - 增加 `PendingSystemOperation`。
  - 增加 `pendingConfirm='system'`。
  - “启动项目”改为确认面板：
    - 操作
    - 目录
    - 原因
    - 影响
    - 将执行命令
    - `[1] 执行一次  [2] 取消`
  - 用户确认后才执行 `npm install` / `npm run dev`。
- `scripts/repl-first-run-smoke.mjs`
  - 覆盖确认面板。
  - 覆盖选择 `1` 后执行并捕获 URL。
- 文档同步：
  - 产品需求文档
  - 架构文档
  - 开发特别声明
  - 开发指南

## 验收标准

- 自然语言“启动项目”不直接执行系统命令。
- 先显示中文确认面板。
- 用户输入 `1` 执行。
- 用户输入 `2` 取消。
- 不出现让用户复制命令的体验。

## 后续扩展

- 抽象 `SystemOperation` 和 `PermissionDecision` 类型。
- 支持“执行并记住此选择”，但只能在低风险且项目级作用域中启用。
- 接入审计日志：每次确认/取消都记录。
- 把验证、安装依赖、提交、回滚、全局记忆写入统一纳入该面板。
