# S5.1 Semantic Intent Router and Chinese REPL UX

## 背景

用户指出两个关键问题：

1. “启动项目”不能只是硬接本地命令，也不能丢给大模型闲聊；应该根据大模型语义、系统上下文和记忆共同判断。
2. 操作上必须假设用户完全不懂，Agent Shell 要聪明地替用户完成，并给出中文结果。
3. REPL 界面需要参考截图风格：中文命令、中文结果、数字选择、执行后明确通过/失败。

## 科学方案

采用混合式语义意图路由：

- 高置信本地动作：当前目录、最近写入位置、启动项目、写入、预览、撤销、状态、诊断，直接由本地工具处理。
- 中置信动作：先展示中文数字选择，让用户确认。
- 高风险动作：删除、覆盖、提交、部署、数据库、密钥、全局记忆、跨项目规则，必须明确授权。
- 复杂语义任务：由大模型生成计划或变更，但输出必须进入系统确认流。
- 记忆参与：用户偏好、项目模板、历史选择作为意图判断信号；全局长期规则不能静默生效。

## 本次实现

- 新增 `doc/DEVELOPER_SPECIAL_DECLARATION.md`，作为 dev1/dev2 后续开发红线。
- `/help` 命令说明中文化。
- 写入、撤销、提交确认选项中文化。
- `/start` / `/serve` 加入 REPL，支持自然语言“启动项目”进入本地执行流。
- `smoke:repl` 覆盖自然语言启动项目：必须执行 dev 脚本、捕获 URL、不触发 AI 聊天。

## 验收标准

- 用户输入“启动项目”时，系统执行启动流程，而不是只给建议。
- 用户输入“你来处理”时，如果上一轮明确是启动项目意图，应继续执行。
- 启动成功后显示访问地址。
- 操作选项使用中文数字。
- `/help` 为中文。
- 完整 smoke 通过。
