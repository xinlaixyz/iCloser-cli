# iCloser Autonomous Execution Chain 自动执行链

日期：2026-05-13
阶段：S6.4

## 1. 目标

这不是给模型看的隐藏思维链，而是给系统执行用的显式任务链。它让 iCloser 在面对完全不懂工程的用户时，可以自动完成：理解目标、找问题、制定计划、中文确认、执行、验证、失败修复、必要时回滚、报告和记忆沉淀。

用户不需要知道命令、目录、测试框架、文件路径或回滚方式。系统负责判断和执行，用户只负责关键授权。


## 1.1 三步任务循环

自动执行链内部必须以三步循环运行：

1. **收集上下文**：模型读取用户目标、当前目录、项目索引、相关文件、错误信息和记忆，形成事实依据。
2. **采取行动**：工具层执行真实动作，例如编辑文件、运行命令、搜索、写文档、生成测试、创建回滚快照。
3. **验证结果**：验证器检查文件、命令、build/lint/test/smoke 和安全规则，失败则把证据送回下一轮。

模型负责思考和推理，工具负责本地动手，验证器负责判断结果。用户可以随时打断、给新指示，或要求“换个方法试试”。流程分支为：用户输入 Prompt → 收集上下文 → 执行操作 → 验证结果；验证通过则任务结束，需要继续则回到收集上下文，用户干预则补充信息、调整方向或中断任务。
## 2. 执行顺序

1. 理解用户目标
   - 输入：用户自然语言、当前目录、记忆、最近会话。
   - 输出：任务意图、风险等级、本地工具需求。
   - 原则：不能把问题直接抛回给用户。

2. 自动检查项目
   - 输入：文件树、依赖、脚本、索引、docs、测试。
   - 输出：项目画像、缺口列表、可运行命令。
   - 原则：只读动作自动执行，不需要确认。

3. 生成小步计划
   - 输入：用户目标、项目画像、缺口列表。
   - 输出：执行步骤、候选文件、验证策略。
   - 原则：大任务必须拆成小步，不能一次性大改。

4. 中文选择确认
   - 输入：执行计划、diff、命令原因、影响。
   - 输出：执行、查看差异、取消、允许并记住。
   - 原则：用户只输入数字，不输入命令。

5. 执行最小变更
   - 输入：用户确认、文件变更契约、系统操作契约。
   - 输出：写入回执、命令输出、变更列表。
   - 原则：写入后必须磁盘验证，告诉用户绝对路径。

6. 自动验证
   - 输入：变更列表、验证配置、测试计划。
   - 输出：build/lint/test/smoke 结果和失败摘要。
   - 原则：不能假装成功，无法验证必须标记 warn。

7. 失败自动修复
   - 输入：失败命令、错误摘要、本轮 diff。
   - 输出：最小修复变更、重试计数。
   - 原则：最多重试 2 次，不扩大修改范围。

8. 安全回滚
   - 输入：写入前快照、任务 diff、失败原因。
   - 输出：回滚结果和诊断报告。
   - 原则：只回滚本任务写入的文件，高风险回滚要确认。

9. 交付报告
   - 输入：变更列表、验证结果、审计事件。
   - 输出：中文报告、文件路径、下一步。
   - 原则：用户要能看懂“做了什么、在哪里、是否成功”。

10. 记忆压缩与沉淀
    - 输入：用户输入事件、任务日志、报告、审计事件。
    - 输出：短期记忆、任务记忆、长期记忆候选。
    - 原则：所有用户输入都进入记忆事件流；长期记忆候选可审核。

## 3. 风险策略

| 动作 | 默认策略 |
| --- | --- |
| 只读分析 | 自动执行 |
| 扫描项目 | 自动执行 |
| 写文件 | 中文选择确认 |
| 运行系统命令 | 中文选择确认 |
| 启动本地服务 | 中文选择确认，可本会话记住 |
| 安装依赖 | 中文选择确认 |
| 删除/覆盖/回滚 | 高风险确认 |
| 自动修复 | 最多 2 次，小步执行 |
| 记忆压缩 | 任务后自动压缩 |

## 4. CLI

```bash
ic autopilot chain
ic auto chain
ic autopilot chain --json
```

JSON 输出：

```json
{
  "version": 1,
  "kind": "autonomous-execution-chain",
  "data": {
    "stages": []
  }
}
```

## 6. 后续开发约束

后续任何“自动写文档、自动写测试、自动修复、自动启动项目”的功能，都必须挂到这条链上，不允许绕开确认、验证、报告和记忆。
