# iCloser Agent Shell 开发特别声明

本声明适用于 dev1、dev2 以及后续加入项目的所有开发者。它不是口号，而是功能验收标准。

## 1. 零知识用户原则

用户可以完全不懂 CLI、Agent、Provider、记忆、Git、依赖安装和项目启动。系统必须替用户完成能确定完成的事情，并把结果说清楚。

- 用户只负责表达目标，例如“启动项目”“补齐文档”“分析代码质量”。
- Agent Shell 负责判断意图、读取当前项目、选择工具、执行动作、展示结果。
- 不允许把工程细节直接抛给用户，例如“建议你运行 npm run dev”，除非系统确实无法安全执行。
- 需要用户参与时，必须给中文数字选项，例如 `[1] 写入  [2] 预览  [3] 取消`。
- 选项必须有推荐默认路径，不能让用户猜下一步。
- 输入框的核心不是让用户输入命令，而是让用户确认系统整理好的选择。

## 2. 聪明 Agent 原则

Agent Shell 不是普通聊天机器人。它运行在用户项目目录里，拥有项目上下文和本地执行能力。

- 能本地读取的信息，不要求用户粘贴。
- 能本地验证的结果，不让用户自己判断。
- 能自动修复的格式问题，不要求用户重说。
- 能记住的偏好，不让用户重复解释。
- 执行后必须告诉用户做了什么、结果如何、文件或服务在哪里。

## 3. 语义意图路由原则

“启动项目”这类需求不能简单等同于硬编码本地命令，也不能全部丢给大模型闲聊。正确方案是混合式语义路由：

1. **高置信本地动作**：当前目录、最近写入位置、启动项目、写入确认、预览、撤销、状态、诊断等，由本地 Intent Router 直接处理。
2. **中置信动作**：系统先给中文选择，让用户确认后执行。
3. **高风险动作**：涉及删除、覆盖、提交、部署、数据库、密钥、全局记忆、跨项目规则时，必须中文确认。
4. **复杂语义任务**：交给大模型生成计划或文件变更，但输出必须进入系统确认流，而不是直接裸展示 JSON。
5. **记忆参与决策**：用户偏好、历史选择、项目模板和团队规则应进入意图判断，但全局长期规则不能静默生效，必须来自明确确认或团队策略。

## 4. UI 与输出原则

界面要参考用户给出的终端截图风格：清楚、中文、可选择、结果明确。

- 命令帮助必须中文。
- 执行过程显示“正在做什么”。
- 执行结果显示“通过 / 失败 / 已取消 / 已写入 / 已启动”。
- 文件写入必须先写入磁盘，再用 `stat` 验证存在；回执显示相对路径、绝对路径和“已确认存在”。
- 服务启动显示访问地址。
- 确认操作使用数字，不要求用户记命令。
- 系统权限操作必须展示确认面板：操作、目录、原因、影响、将执行的命令、数字选择。
- 不展示给普通用户内部 JSON、内部状态机和英文术语堆叠。

## 5. 验收红线

以下行为视为 bug：

- 用户问当前路径，系统让用户自己提供路径。
- 用户说“启动项目”，系统只建议命令但不执行。
- 用户确认写入后，系统不告诉文件写到哪里。
- 系统显示写入成功，但磁盘上没有对应文件，或没有做写后存在性验证。
- 用户看到选择项后，输入 `1和2` 被当成普通聊天。
- 用户看到英文命令解释而不知道下一步。
- Agent 说“我无法访问你的文件系统”，但实际运行在项目目录里。
- 系统权限操作直接执行，未先展示中文确认面板和数字选择。

## 6. 生产环境 API Key 红线

**正式环境 API Key 是项目根基，所有开发者必须遵守。**

### 绝对禁止

- ❌ 修改、删除、注释、移动 `~/.icloser/config.json` 中已配置的 `ai.apiKey`
- ❌ 在项目文件中硬编码真实 API Key（包括但不限于 `.ts` `.js` `.json` `.md` `.mjs`）
- ❌ 将包含真实 Key 的配置文件提交到 Git
- ❌ 在 CI/CD 日志、错误输出、调试打印中暴露 Key（系统已内置 `maskApiKey`，必须使用）
- ❌ 在测试用例、smoke 脚本、示例代码中使用真实 Key（只能用占位符 `sk-xxx` 或假 Key `sk-1234...`）
- ❌ 将真实 Key 写入 Claude Code 的 `settings.local.json` 权限白名单

### 必须遵守

- ✅ API Key 只存放在两个地方：
  1. **环境变量**：`DEEPSEEK_API_KEY` / `ANTHROPIC_API_KEY` / `OPENAI_API_KEY`（生产环境推荐）
  2. **全局配置**：`~/.icloser/config.json`（通过 `/apikey` 命令写入，本地开发用）
- ✅ 代码中读取 Key 必须走 `resolveApiKeyForProvider()`，不允许直接读 `process.env`
- ✅ 日志/错误信息中输出 Key 必须经过 `maskApiKey()` 脱敏（如 `sk-2b90...29b0`）
- ✅ `.gitignore` 必须包含 `.env`、`.claude/settings.local.json`、`*.local.json`
- ✅ 切换 Provider 后必须运行 `ic provider test` 验证连接正常

### 泄露处理

一旦发现真实 Key 出现在不该出现的地方：
1. 立即删除泄露的 Key
2. 到对应平台（DeepSeek/Anthropic/OpenAI）重置 Key
3. 检查是否有未授权调用记录
4. 更新 `.gitignore` 和文档防止再次发生
