# AgentCode — UI/UX 设计文档

版本：0.2.0
日期：2026-05-15

## 设计原则

```
清晰度 > 装饰性    信息层级清楚，不乱
反馈感 > 沉默       每个动作都有响应，等待有进度
可达性 > 记忆       常用操作一键可达，减少输入
容错性 > 严格       错误有指引，操作可撤销
```

## 1. 输入系统

### 1.1 双线输入框

输入区域以 Unicode 双线框围住，视觉上区分「输入区」和「对话区」。

**默认状态 (单行):**

```
  ╭─ 输入 ───────────────────────────────────────────╮
  │                                                  │
  │  用户输入内容                                     │
  │  █                                               │
  │                                                  │
  ╰─ Enter 发送  │  Shift+Enter 换行  │  ↑↓ 历史 ────╯
```

**多行输入 (Shift+Enter 换行后):**

```
  ╭─ 输入 ───────────────────────────────────────────╮
  │                                                  │
  │  帮我把数据库连接改成连接池模式                    │
  │  具体要求：                                       │
  │  1. 最小连接数 5，最大 20                         │
  │  2. 超时 30 秒                                   │
  │  █                                               │
  │                                                  │
  ╰─ Enter 发送  │  Shift+Enter 换行  │  ↑↓ 历史 ────╯
```

**规则:**
- `Enter` → 发送当前输入内容
- `Shift+Enter` → 插入换行符 `\n`
- 输入框最小高度 3 行，最大高度 6 行 (超出滚动，顶部显示 `↑ 还有 N 行`)
- 发送后框自动收起到单行

### 1.2 历史导航

- `↑` / `↓` 在输入框内切换历史记录
- `Ctrl+R` 打开历史搜索面板，实时过滤

### 1.3 输入校验

- 输入 `/` 后实时提示命令自动补全
- 未知命令灰色斜体显示

## 2. 变更预览 (Diff 着色)

### 2.1 代码变更

```
  ╭─ 变更预览 ───────────────────────────────────────╮
  │                                                  │
  │  src/config.ts                                   │
  │  ───────────────────────────────────────────     │
  │   1  import { Pool } from 'pg-pool';             │
  │ ─ 2  const db = { single: true }                  │
  │ + 2  const pool = new Pool({ min: 5, max: 20 })   │
  │                                                  │
  ╰──────────────────────────────────────────────────╯
```

**着色规则:**
- `─` 删除行：红底/红色前景，行首 `─` 标记
- `+` 新增行：绿底/绿色前景，行首 `+` 标记
- 未变更行：灰色，不带行首标记

### 2.2 文档变更

同样规则适用于 Markdown 文档：

```
  ─ ## 旧标题        (红色)
  + ## 新标题        (绿色)
  ─ 已废弃的说明      (红色)
  + 新增的说明        (绿色)
```

### 2.3 全局启用

- REPL `/diff` 命令：预览当前 pending files
- CLI `ic d <task-id>` 命令：查看任务 diff
- 任务报告中的 diff 区块

## 3. 输出消毒

### 3.1 过滤规则

所有进入 `process.stdout` 的输出经过 `sanitizeOutput()` 过滤器：

| 字符范围 | 处理方式 |
|---------|---------|
| ASCII 0-8 | 丢弃 |
| ASCII 11-12 | 丢弃 |
| ASCII 14-31 | 丢弃 |
| \n (10), \r (13), \t (9) | 保留 |
| 未闭合 ANSI CSI 序列 | 丢弃序列 |
| 超长行 (>1000字符) | 截断 + `…(截断)` |
| � 替换字符 | 丢弃 |
| 连续 3+ 相同非 ASCII 字符 | 替换为 `�` |

### 3.2 集成点

- `printChatChunk()` — 流式 AI 输出逐行输出
- `process.stdout.write` — 全局兜底包装
- 启动时打印 "输出过滤器已启用"

## 4. 等待体验

### 4.1 三阶段反馈

**阶段 1 — 连接 (0-1s):**
```
  ◉ 正在连接 DeepSeek API...
```

**阶段 2 — 等待首字节 (1-15s):**
```
  ◉ AI 分析中 [4.7s]  ██████░░░░░░░░░░░░
```

- 计时器每秒更新 `[N.Ns]`
- 脉冲动画：`◉` `◔` `◑` `◕` 循环，每 80ms 切换
- 超过 10s 显示提示：`复杂任务可能需 10-30 秒，Ctrl+C 可中断`

**阶段 3 — 流式输出:**
```
  ◉ 输出中 [8.1s]  已接收 847 tokens

  好的，我来帮你修改数据库配置。首先分析...
  █  (内容持续追加)
```

### 4.2 Token 计数

从 AI 响应中实时提取 `x-tokens` 或从 stream chunk 估算，每秒刷新一次。

### 4.3 提前呈现

- 不等到完整 code block 才渲染
- 逐行从 stream buffer 中取出，即时打印
- 代码块识别到 ``` 开启/关闭时分别渲染边框

## 5. 底部固定操作面板

### 5.1 空闲状态

底部常驻快捷命令栏：

```
  ╭─ 操作 ───────────────────────────────────────╮
  │  help  帮助  │  scan  扫描  │  status  状态    │
  │  write 写入  │  diff  预览  │  clear  清屏     │
  │  Ctrl+P 命令面板                              │
  ╰───────────────────────────────────────────────╯
```

按键规则：输入框为空时，按对应字母直接触发操作。有输入内容时字母进入输入框，不触发操作。

### 5.2 有待写入文件时

```
  ╭─ 待处理文件 (3) ───────────────────╮
  │ ● src/config.ts             +12 行  │
  │ ● src/types.ts               +5 行  │
  │ ● docs/README.md             +3 行  │
  ├─ 操作 ──────────────────────────────┤
  │ [w] 写入全部  [d] 预览变更  [x] 取消│
  ╰─────────────────────────────────────╯
```

- `w` → 写入全部文件
- `d` → 打开 diff 预览
- `1/2/3` → 选择/取消单个文件
- `x` → 清空 pending files

### 5.3 AI 执行中

```
  ╭─ 状态 ────────────────────────────────────────╮
  │ ◉ AI 执行中 [12.3s]  任务: 修改数据库配置       │
  │ Ctrl+C 中断                                     │
  ╰─────────────────────────────────────────────────╯
```

### 5.4 验证完成

```
  ╭─ 验证通过 ──────────────────────────────────────╮
  │ ● tsc 编译    ✓ 0 errors                       │
  │ ● eslint      ✓ 0 warnings                     │
  │ ● vitest      ✓ 18/18 passed                   │
  ├─ 操作 ──────────────────────────────────────────┤
  │ [c] 提交  [u] 撤销  [r] 查看报告                │
  ╰─────────────────────────────────────────────────╯
```

### 5.5 验证失败

```
  ╭─ 验证失败 (2/4 通过) ──────────────────────────╮
  │ ● tsc 编译    ✓ 通过                           │
  │ ● eslint      ✗ 2 warnings                     │
  │ ● vitest      ✓ 通过                           │
  │ ● e2e         ✗ 超时                           │
  ├─ 操作 ──────────────────────────────────────────┤
  │ [f] AI 自动修复  [u] 撤销变更  [i] 忽略继续     │
  ╰─────────────────────────────────────────────────╯
```

### 5.6 技术实现

- 利用现有 `TuiScreen` 类 (src/cli/tui.ts) 扩展
- 底部面板控制序列 `\x1b[s` 保存光标，渲染面板，`\x1b[u` 恢复
- 面板行数动态计算：内容区 = 终端高度 - 面板高度 - 输入框高度
- SIGWINCH 信号监听窗口大小变化，自动重绘
